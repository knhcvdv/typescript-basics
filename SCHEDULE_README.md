# Система управління розкладом в університеті

## Опис

Система для управління розкладом занять в університеті, реалізована з використанням TypeScript концепцій:
- **Union Types** - для обмеження можливих значень
- **Type Aliases** - для створення переіспользуваних типів
- **Arrays** - для зберігання та обробки даних

## Структура проекту

- `src/schedule.ts` - основна логіка системи з усіма типами та функціями
- `src/schedule-demo.ts` - демонстрація роботи системи з прикладами

## Реалізовані функції

### 1. Визначення базових типів
- ✅ `DayOfWeek` - дні тижня (Monday-Friday)
- ✅ `TimeSlot` - часові слоти занять
- ✅ `CourseType` - типи занять (Lecture, Seminar, Lab, Practice)

### 2. Основні структури
- ✅ `Professor` - інформація про професора
- ✅ `Classroom` - інформація про аудиторію
- ✅ `Course` - інформація про курс
- ✅ `Lesson` - інформація про заняття

### 3. Робота з масивами
- ✅ `addProfessor()` - додавання професора
- ✅ `addLesson()` - додавання заняття з перевіркою конфліктів

### 4. Пошук та фільтрація
- ✅ `findAvailableClassrooms()` - пошук вільних аудиторій
- ✅ `getProfessorSchedule()` - розклад професора

### 5. Валідація та конфлікти
- ✅ `validateLesson()` - перевірка конфліктів розкладу
- ✅ `ScheduleConflict` - тип для опису конфліктів

### 6. Аналіз та звіти
- ✅ `getClassroomUtilization()` - відсоток використання аудиторії
- ✅ `getMostPopularCourseType()` - найпопулярніший тип занять

### 7. Модифікація даних
- ✅ `reassignClassroom()` - зміна аудиторії для заняття
- ✅ `cancelLesson()` - видалення заняття з розкладу

## Запуск

```bash
# Компіляція TypeScript
npm run build

# Запуск демонстрації
node dist/schedule-demo.js
```

## Особливості реалізації

- Всі типи визначені через type aliases та union types
- Повна типізація всіх функцій та параметрів
- Ефективна робота з масивами (filter, find, map)
- Обробка крайових випадків та помилок
- Валідація конфліктів (професор/аудиторія)
- Коментарі пояснюють логіку складних частин
